CS 360 Module Eight Journal
App Requirements and Goals
The mobile app I developed for this course was designed to meet a clear user need by providing a simple and reliable way for users to log in, manage data, and receive important notifications through an intuitive interface. The goal was to create an app that balanced functionality with usability, ensuring that users could complete tasks quickly and without confusion.

User Needs, Screens, and UI Design
To support these needs, the app included several key screens such as a login screen, a home dashboard, data-entry screens, and a settings area. Each screen was intentionally designed with user-centered principles in mind, using clean layouts, consistent navigation, and clear labels to reduce cognitive load and help users move through the app naturally.

Coding Approach and Strategies
When coding the app, I approached the process by breaking the work into small, manageable components. I focused on building one feature at a time, testing it, and then integrating it with the rest of the system. I used strategies such as modular coding, consistent naming conventions, and incremental testing to keep the project organized and maintainable. These techniques will be valuable in future development work because they simplify debugging, reduce errors, and make it easier to expand or modify an app later.

Testing and Code Validation
Testing played a major role in ensuring the app was functional. I tested each feature as I built it, especially components involving permissions, database operations, and SMS notifications. This process revealed issues early, such as permission handling problems and data-flow inconsistencies, allowing me to fix them before they became larger obstacles.

Challenges and Innovation
Throughout the full design and development process, I encountered several challenges that required creative problem-solving. One of the biggest areas where I had to innovate was managing the interaction between the UI and the backend logic, especially when dealing with Android permissions and lifecycle behaviors. Ensuring that the app behaved consistently across different states required careful planning and testing.

Areas of Success
Despite these challenges, I was particularly successful in demonstrating my skills in connecting the UI to functional backend features. The login system, database integration, and notification functionality all showcased my ability to combine user-centered design with practical mobile development techniques. This project strengthened my technical skills and reinforced the importance of thoughtful planning, iterative development, and user-focused design.
